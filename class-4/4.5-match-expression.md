# 4.5 The `match` Expression

In this video, we cover the `match` expression, a newer control structure
introduced in PHP 8. It provides a concise way to map a value to a result.

## Match vs. Switch

Use `match` when you want an expression that returns a value and you want
strict comparisons. Use `switch` when you need fall-through behavior or want
side effects in each case.

Key differences:

- `match` uses strict comparison (`===`).
- `match` returns a value.
- `match` does not allow fall-through.
- `match` requires every case to be explicit (use `default` when needed).

## Basic Match Example

```php
<?php
$day = 'Tuesday';

$message = match ($day) {
    'Monday' => 'Start of the workweek!',
    'Tuesday' => 'It\'s Tuesday!',
    'Wednesday' => 'Midweek already.',
    default => 'Enjoy your day!',
};

print $message . PHP_EOL;
?>
```

## Matching on Types

Because `match` uses strict comparison, type differences matter.

```php
<?php
$input = '1';

$result = match ($input) {
    1 => 'integer 1',
    '1' => 'string 1',
    default => 'other',
};

print $result . PHP_EOL;
?>
```

## Grouping Multiple Values

You can map multiple values to the same result.

```php
<?php
$status = 201;

$message = match ($status) {
    200, 201 => 'Success',
    400, 401, 403 => 'Client error',
    500 => 'Server error',
    default => 'Unknown status',
};

print $message . PHP_EOL;
?>
```

## When Match Is Useful

`match` is great for mapping known inputs to outputs, such as labels,
messages, and display text. Use it when you want a clear, value-returning
expression instead of a longer `switch` block.

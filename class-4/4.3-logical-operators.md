# 4.3 Logical Operators

In this video, we take a deeper look at logical operators. These combine or
negate boolean expressions and always produce a boolean result. That result
can be stored in a variable, passed to a function, or used directly in a
control structure.

## AND (`&&` and `and`)

AND is true only when both sides are true. PHP supports both `&&` and the
`and` keyword.

```php
<?php
$isMember = true;
$hasPaid = false;

$canAccess = $isMember && $hasPaid;

// Note: Unlike `&&`, `and` has lower precedence than `=`
// so this requires parentheses to work as expected.
$canAccessAlt = ($isMember and $hasPaid); 

if ($isMember and $hasPaid) {
    print 'Access granted.' . PHP_EOL;
}
?>
```

## OR (`||` and `or`)

OR is true when at least one side is true. PHP supports both `||` and the
`or` keyword.

```php
<?php
$isAdmin = false;
$isEditor = true;

$hasAccess = $isAdmin || $isEditor;

// Note: Operator precedence means you need parentheses with
// `or` to get the expected result as well.
$hasAccessAlt = ($isAdmin or $isEditor);

if ($isAdmin || $isEditor) {
    print 'Access granted.' . PHP_EOL;
}
?>
```

## NOT (`!`)

NOT flips a boolean value. PHP supports the `!` operator.

```php
<?php
$isLoggedIn = false;

$needsLogin = !$isLoggedIn;

if (!$isLoggedIn) {
    print 'Please sign in.' . PHP_EOL;
}
?>
```

## XOR (`xor`)

XOR is true when exactly one side is true. PHP supports the `xor` keyword.

```php
<?php
$hasCoupon = true;
$isEmployee = false;

$discountEligible = ($hasCoupon xor $isEmployee);

if ($hasCoupon xor $isEmployee) {
    print 'Apply discount.' . PHP_EOL;
}
?>
```

## Boolean Results in Practice

Logical expressions always return a boolean. You can store the result or pass
it to a function just like any other value.

```php
<?php
$age = 20;
$isMember = true;

$eligible = $age >= 18 && $isMember;

var_dump($eligible); // bool(true)

if ($eligible) {
    print 'Eligible.' . PHP_EOL;
}
?>
```

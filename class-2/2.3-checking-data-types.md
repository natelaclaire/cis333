# 2.3 Checking Data Types

In this video, we will learn how to check data types using the `is_*` functions and other helpful tools like `gettype`, `var_dump`, and strict comparisons.

PHP provides a family of `is_*` functions to check types directly. These return `true` or `false`.

```php
<?php
$age = 20;
$price = 19.99;
$name = "Ada";
$isActive = true;
$tags = ["php", "web"];

var_dump(is_int($age));      // true
var_dump(is_float($price));  // true
var_dump(is_string($name));  // true
var_dump(is_bool($isActive));// true
var_dump(is_array($tags));   // true

var_dump(is_numeric("42"));  // true
var_dump(is_numeric("4.2")); // true
var_dump(is_numeric("42a")); // false
?>
```

The `ctype_*` functions check character data in strings. They are useful when you want to validate input that should be all digits or all letters.

```php
<?php
var_dump(ctype_digit("123"));  // true
var_dump(ctype_alpha("abc"));  // true
var_dump(ctype_digit("12a"));  // false
var_dump(ctype_alnum("abc1")); // true - all alphanumeric
?>
```

There are several other `ctype_*` functions for different character checks, which you can explore in the [PHP documentation](https://www.php.net/manual/en/book.ctype.php).

`gettype` returns the type name as a string. `var_dump` prints both the type and the value, along with size details for strings and arrays, which makes it great for debugging.

```php
<?php
$value = 42;

echo gettype($value); // integer
var_dump($value);     // int(42)
?>
```

It's important to understand the difference between `isset` and `is_null`. `isset` checks if a variable is defined and not NULL, while `is_null` checks if a variable is explicitly set to NULL.

```php
<?php
// Unset variable
// $color is never defined
var_dump(isset($color));   // false
var_dump(is_null($color)); // true (with a notice if not defined)

// Explicit NULL
$color = null;
var_dump(isset($color));   // false
var_dump(is_null($color)); // true
?>
```

Additionally, PHP provides an `empty` function that checks if a variable is not set or has a "falsy" value like 0, "", or NULL.

```php
<?php
//$var0 is never defined
$var1; // not set
$var2 = null;
$var3 = 0;
$var4 = "";
$var5 = "Hello";
var_dump(empty($var0)); // true
var_dump(empty($var1)); // true
var_dump(empty($var2)); // true
var_dump(empty($var3)); // true
var_dump(empty($var4)); // true
var_dump(empty($var5)); // false
?>
```

You can also check type by using strict comparisons. The `===` operator checks both value and type, while `==` allows type juggling, which we'll dig more into in the next video.

```php
<?php
$count = 5;
$text = "5";

var_dump($count == $text);  // true
var_dump($count === $text); // false
?>
```

Use `is_*` functions for clear type checks, `gettype` and `var_dump` for debugging, and strict comparisons to avoid type juggling.
